# End-of-day report (G30 increment 1)

Status: **SHIPPED 2026-07-14** (review code-reviews/2026-07-14-end-of-day-report.md). Farshid: "report generator on
the background… like end of the day". The classic Z-report, generated by
the till itself.

## Report content (one business day)

Sales count + gross, refunds count + amount, **net**, tax (sales −
refunds), takings per payment method (in / refunded out / net), receipt
number range, generated-at stamp. Data comes from the same repos the
Reports page and ask-your-till already use.

## Generation paths

1. **Manual**: "End-of-day report" button on the Reports page
   (manager/admin) — generates, archives, prints.
2. **Scheduled**: settings `reports.eod_time` (HH:MM) + enable toggle
   (manager card on Settings). A background loop (in `pages.Init`, where
   the printer/settings plumbing lives) checks ~every 30s: past the
   configured time and no report archived for today → generate, archive,
   print. A till that's off at closing time catches up at next boot
   (the check is "today's report missing", not "it is exactly 21:30").
3. Printed on the receipt printer when one is configured (80mm Z-report);
   always archived regardless.

## Archive

Migration 013 `report_archive` (kind, period, content JSON, created_at;
unique kind+period). Reports page lists recent archived reports with
view + reprint. Email/push lands with cloud services (G30 later);
`report`/`scheduler` plugin types remain the extension point for custom
reports.
