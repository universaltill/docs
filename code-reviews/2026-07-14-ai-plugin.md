# Review — AI as an opt-in plugin (P2.1) + generic plugin settings editor

Date: 2026-07-14 · Repos: universal-till, **ut-plugin-integration-ai**
(new) · Spec: `architecture/ai-plugin.md` (written first). Implements the
2026-07-14 direction: AI is a plugin, not active by default.

## What shipped

- **`ut-plugin-integration-ai` v1.0.1** (`runtime: none`,
  `canonical_type: integration` — ships no code; it is the opt-in grant):
  manifest settings `endpoint`/`vision_model`/`ask_model` seed
  `plugin_settings` on install. Listing
  `884b3515-88e5-4e66-903e-2f622a3a58dd`. (v1.0.0 taught us the
  marketplace requires ≥1 permission scope → declares `ai.configure`.)
- **Host resolution per request** (`aiService`): plugin active → Ollama
  config from its settings; else `UT_AI_*` env (dev override); else
  disabled. `Deps.AI` is now purely a test seam — `pages.Init` no longer
  builds a service from env. Two tiny queries on AI routes only; never on
  the sale path; changes apply with no reload.
- **Generic plugin settings editor** (the reusable piece G23 terminal
  plugins will use): `/plugins/{id}/settings` + `POST
  /api/plugins/{id}/settings` (manager, audited) — only manifest-declared
  keys are writable; Settings button on the plugins page. Repo:
  `ListPluginSettings` / `UpsertPluginSetting` (update-then-insert — the
  unique index includes NULL `scope_id`, which SQLite treats as distinct,
  so ON CONFLICT can't be trusted) / `PluginActive`.

## Bugs caught live (fixed)

1. `ListPluginSettings` scanned SQLite's TEXT `updated_at` into
   `time.Time` — the driver refused, and because the resolver treats a
   settings error as "no config", **everything silently fell back to
   disabled**. Fixed (string scan + parse); the E2E caught it as
   "installed but still invisible".
2. Working-directory slip during repo scaffolding briefly overwrote
   universal-till's `.gitignore` and dropped a stray file — restored from
   git before any commit; no damage landed.

## Verification

Full live cycle on the dev marketplace, env vars unset: **no plugin →
ask 404 + no camera button** → one-click install ("AI Assistant
(self-hosted) v1.0.1", settings seeded) → camera button appears → edited
`ask_model` to qwen3.6 in the new settings editor ("applied immediately")
→ **real Ask answered "13 sales today." through the plugin-configured
endpoint** → disable → invisible again. Full suite + guards green.
